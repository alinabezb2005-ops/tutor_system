## Что входит в систему

```
tutor_system/
├── main.py              ← сервер (FastAPI) + Telegram бот
├── requirements.txt     ← зависимости Python
├── Procfile             ← для хостинга Railway
├── static/
│   ├── cabinet.html     ← кабинет ученика (главная страница)
│   ├── admin.html       ← панель репетитора
│   └── uploads/         ← загружаемые файлы (создаётся автоматически)
└── data/
    ├── students.json    ← база учеников (создаётся автоматически)
    ├── knowledge.json   ← база знаний (создаётся автоматически)
    └── programs.json    ← программы обучения (создаётся автоматически)
```

**Сайт работает так:**
- `твой-сайт.railway.app/` → кабинет ученика
- `твой-сайт.railway.app/admin` → панель репетитора
- Telegram бот → те же данные, та же база

## Шаг 1 — Настройка

Открой `main.py` и найди строки вверху:

```python
BOT_TOKEN      = os.getenv("BOT_TOKEN", "ВСТАВЬ_ТОКЕН_СЮДА")
ADMIN_ID       = int(os.getenv("ADMIN_ID", "0"))
ADMIN_PASSWORD = os.getenv("ADMIN_PASSWORD", "ilya2024")
```

**BOT_TOKEN** — токен от @BotFather (если бот не нужен — оставь как есть)
**ADMIN_ID** — твой Telegram ID (узнать: написать @userinfobot → `/start`)
**ADMIN_PASSWORD** — пароль для входа в панель репетитора (измени на свой!)

## Шаг 2 — Локальный запуск (для теста)

### Установка Python (если нет)
Скачай с **python.org/downloads**, при установке поставь галочку "Add to PATH"

### Установка зависимостей
```bash
cd tutor_system
pip install -r requirements.txt
```

### Запуск сервера
```bash
uvicorn main:app --reload
```

Открой в браузере:
- **http://localhost:8000** → кабинет ученика
- **http://localhost:8000/admin** → панель репетитора

## Шаг 3 — Деплой на Railway (бесплатный хостинг)

### 3.1 Создай аккаунт
Зайди на **railway.app** → Sign up with GitHub

### 3.2 Загрузи код на GitHub
1. Зайди на **github.com** → New repository → назови `tutor-cabinet`
2. Скачай GitHub Desktop (desktop.github.com) — проще всего
3. Clone → выбери репозиторий
4. Перекопируй все файлы `tutor_system/` в папку репозитория
5. Commit → Push

### 3.3 Подключи к Railway
1. Railway → New Project → Deploy from GitHub repo
2. Выбери `tutor-cabinet`
3. Railway автоматически обнаружит `Procfile` и запустит

### 3.4 Добавь переменные окружения
В Railway → Settings → Variables добавь:

| Переменная | Значение |
|---|---|
| `BOT_TOKEN` | токен от @BotFather |
| `ADMIN_ID` | твой Telegram ID |
| `ADMIN_PASSWORD` | придуманный пароль |

### 3.5 Готово!
Railway даст ссылку вида: `https://tutor-cabinet-production.up.railway.app`

Это и есть адрес твоего сайта!

## Как пользоваться системой

### Добавить ученика

**Через панель репетитора** (удобнее):
1. Открой `/admin`
2. Введи пароль репетитора
3. Вкладка «Добавить» → заполни имя, пароль, программу → «Создать кабинет»

**Через Telegram бот**:
```
/добавить Аня Иванова; 1234; Математика ЕГЭ профиль
```

### Выдать задание

**Панель** → «Задания» → выбери ученика, заполни, нажми «Выдать»

**Telegram**:
```
/задание Аня Иванова; Алгебра; §12 упр 5-9; 2026-04-25
```

### Поставить оценку

**Telegram**:
```
/оценка Аня Иванова; Алгебра; Производные; 5; Отличная работа!
```

### Добавить занятие в расписание

**Telegram**:
```
/занятие Аня Иванова; Понедельник 20 апреля; 16:00 – 17:30; Алгебра; Интегралы
```

### Обновить прогресс

**Панель** → «Прогресс» → выбери ученика → двигай ползунки → «Сохранить»

**Telegram**:
```
/прогресс Аня Иванова; Производные; 75
```

### Загрузить материал в базу знаний

**Панель** → «База знаний» → выбери тип (текст/ссылка/файл) → заполни → «Загрузить»

**Telegram** (только ссылки и тексты):
```
/знание Алгебра; Видео: Производные; https://youtube.com/watch?v=...
```

### Написать ученику

**Telegram**:
```
/сообщение Аня Иванова; Не забудь подготовиться к контрольной!
```

## Как ученик заходит в кабинет

1. Открывает ссылку на сайт (например: `tutor-cabinet.railway.app`)
2. Вводит своё имя **точно как у репетитора** (например: `Аня Иванова`)
3. Вводит пароль (ты устанавливаешь при создании)

Кабинет обновляется каждые 8 секунд — ученик сразу видит новые задания и сообщения.

## Если что-то не работает

**Сайт не открывается** → проверь статус деплоя в Railway (должно быть зелёное «Active»)

**Бот не отвечает** → проверь правильность BOT_TOKEN и ADMIN_ID в переменных Railway

**Ученик не может войти** → имя должно совпадать точно, включая регистр и пробелы

**Файлы не сохраняются на Railway** → Railway сбрасывает файловую систему при редеплое. Для постоянного хранения файлов нужно подключить внешнее хранилище (Cloudinary или S3). Обратись — помогу настроить.
